### constitution.md
**项目宪章** - 版本 v1.1.0

完整的项目治理原则和开发规范，包括：

- **核心原则**：11 条强制执行的开发原则
  - I. 核心技术栈要求（Python 3.X）
  - II. 代码质量门禁（99%+ 类型覆盖率，ruff 格式化，标准 Docstring）
  - III. 架构完整性（DDD，禁止循环依赖）
  - IV. 响应一致性（标准化响应结构）
  - VIII. 测试先行（TDD 红-绿-重构）
  - IX. 文档语言规范（代码英文，文档中文）
  - X. 文件操作闭环（原地修改，禁止过度新建）
  - XI. Git 提交规范（格式化 → 质量检查 → Conventional Commits → 明确 push）

- **实施要求**：设计阶段、开发阶段、代码审查阶段的强制检查点

- **治理规则**：宪章修订流程、版本控制策略、合规审查

**非妥协原则**（标记为"非妥协"的原则违反需要技术负责人书面批准）：

## 快速参考

2. **TDD 流程**：红-绿-重构循环

4. **文档规范**：代码用英文，文档用中文
5. **文件操作**：原地修改，避免创建新版本文件
